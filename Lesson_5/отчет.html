BDU:2022-06576 Уязвимость в Python до 3.9.16 (python-3916)
Описание уязвимости
Python 3.9.x до 3.9.16 и 3.10.x до 3.10.9 в Linux допускает локальное повышение привилегий в конфигурации, отличной от конфигурации по умолчанию. Библиотека многопроцессорной обработки Python при использовании с методом запуска forkserver в Linux позволяет десериализовать pickles от любого пользователя в локальном сетевом пространстве имен того же компьютера, что во многих системных конфигурациях означает любого пользователя на том же компьютере. Pickles могут выполнять произвольный код. Таким образом, это допускает локальное повышение привилегий пользователя до пользователя, от имени которого запущен любой процесс forkserver. Установка multiprocessing.util.abstract_sockets_supported в значение False является обходным путем. Метод запуска forkserver для многопроцессорной обработки не является методом запуска по умолчанию. Эта проблема характерна для Linux, поскольку только Linux поддерживает сокеты абстрактного пространства имен. CPython до версии 3.9 по умолчанию не использует сокеты абстрактного пространства имен Linux. Поддержка ручного указания пользователями сокета абстрактного пространства имен была добавлена ​​в качестве исправления ошибки в версиях 3.7.8 и 3.8.3, но для этого в CPython до версии 3.9 пользователям необходимо было выполнять особые необычные вызовы API.

Однако реализация функции не включает проверку, что позволяет сертификатам с недействительными или неправильными политиками проходить проверку сертификата. Проверки политики сертификата по умолчанию отключены в OpenSSL и обычно не используются приложениями.

BDU:2019-03123 Уязвимость в OpenSSL 1.1.1-1.1.1c, 1.1.0-1.1.0k и 1.0.2-1.0.2s (20190730)
Описание уязвимости
OpenSSL имеет внутреннее значение по умолчанию для дерева каталогов, где он может найти файлы конфигурации и сертификаты. Этот каталог обычно называется OPENSSLDIR и настраивается с помощью параметров --prefix / --openssldir. OpenSSL версий 1.1.1-1.1.1c, 1.1.0-1.1.0k и 1.0.2-1.0.2s для mingw конфигурации предполагают, что результирующие программы и библиотеки установлены в Unix-подобной среде и префикс по умолчанию задан "/usr/local". Тем не менее, mingw-это программы Windows, и поэтому они ищут подкаталоги 'C:/usr/local', который может быть общедоступен для записи, что позволяет злоумышленникам изменять конфигурацию OpenSSL, вставлять сертификаты CA, изменять (или даже заменять) существующие модули и т.д.
Возможные меры по устранению уязвимости
Использование рекомендаций: Для OpenSSL: https://www.openssl.org/news/secadv/20190730.txt Для РЕД ОС: Обновление операционной системы до версии 7.2 Муром Для Fedora: https://lists.fedoraproject.org/archives/list/package-announce@lists.fedoraproject.org

BDU:2024-05196 Уязвимость в Python с 3.8.20 до 3.11.10 (python-3820)
Описание уязвимости
Модуль «ipaddress» содержал неверную информацию о том, были ли определенные адреса IPv4 и IPv6 обозначены как «глобально достижимые» или «частные». Это повлияло на свойства is_private и is_global классов ipaddress.IPv4Address, ipaddress.IPv4Network, ipaddress.IPv6Address и ipaddress.IPv6Network, где значения не возвращались в соответствии с последней информацией из реестров адресов специального назначения IANA. CPython 3.12.4 и 3.13.0a6 содержат обновленную информацию из этих реестров и, таким образом, имеют предполагаемое поведение.
